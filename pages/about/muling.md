# 研究对象：慕棱

如果在一份正经的产品说明书里，这里应该写着“系统架构师”或者“DevOps 工程师”。
但在这里，我们只称他为：**赛博野生包工头**。

## 基础属性配置

- **职业**：表面上是个在景德镇搞陶瓷的大学生，背地里是个天天跟各种代理、API 和 Docker 容器过不去的黑户网管。
- **技术流派**：主打一个“面向报错编程”和“缝合怪架构”。没有任何科班规矩的束缚，能用 Nginx 反代出来的绝对不写后端，能白嫖的免费套餐绝对不花一分钱（买香港节点和洛天依周边除外）。
- **特长**：徒手搓全栈、肉身抗风控、在茫茫盲盒中一发入魂抽出 0.57% 的隐藏款。
- **致命弱点**：无法忍受丑陋的默认 UI 面板。

## 为什么他总在折腾？

“因为我需要掌控感。”
这是他最常挂在嘴边的话。

其实翻译成白话就是：他不相信任何黑盒产品。与其抱怨哪个软件用着不爽，不如自己搭一套。从底层逻辑到界面颜色，所有的东西必须按照他的强迫症标准来。

他可以为了把 NewAPI 的默认界面改成洛天依主题，熬夜和前端代码死磕；也可以为了让几个 OpenAI 的号多活两天，每天早起跟各大平台的封控机制斗智斗勇。

这不是在搞什么伟大的商业闭环，这纯粹就是一个技术宅的固执。
