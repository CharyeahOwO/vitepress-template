# 环境变量

环境变量用于控制 New API 的运行行为。不同版本支持的变量可能不同，下面整理的是常见配置思路。

## 基本原则

- 敏感信息不要写进公开仓库；
- 生产环境不要使用临时密码；
- 改环境变量后通常需要重启容器；
- 修改前先备份当前配置。

## 常见类别

### 运行环境

用于设置时区、监听端口、运行模式等。

```text
TZ=Asia/Shanghai
```

### 数据库配置

如果使用外部数据库，需要配置连接信息。数据库地址、用户名和密码都属于敏感信息，不要公开。

### Redis 配置

部分部署会使用 Redis 做缓存、队列或限流辅助。Redis 不可用时，可能导致部分功能异常。

### 安全相关

包括管理员初始化、会话密钥、访问控制等配置。

这类变量一旦泄露，可能导致后台被接管或数据暴露，务必放在服务器本地环境或容器编排平台的 Secret 中。

## 修改流程

推荐流程：

```text
备份当前配置
  ↓
修改环境变量
  ↓
重启服务
  ↓
查看日志
  ↓
访问后台确认功能
```

如果修改后服务无法启动，先回滚到上一个可用配置，再看日志定位问题。

## 不建议的做法

不要把包含密钥的 `.env` 文件提交到 GitHub。

不要把上游 API Key 写进文档示例。

不要把生产环境和测试环境混用同一套密钥。
