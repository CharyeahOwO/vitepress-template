# 升级与备份

升级 New API 之前，先备份。别等容器起不来时才想起数据还没存档。

## 升级前检查

升级前建议确认：

- 当前版本；
- 镜像来源；
- 数据库类型；
- 数据目录位置；
- 反代配置；
- 是否有自定义前端或本地修改。

如果你改过前端、主题或源码，不要直接覆盖升级，先确认改动是否需要重新合并。

## 备份内容

至少备份：

- 数据库；
- 挂载的数据目录；
- `docker-compose.yml`；
- `.env` 或环境变量记录；
- 反向代理配置；
- 自定义前端文件。

## Docker 升级流程

常见流程：

```bash
docker compose pull
docker compose up -d
docker compose logs -f
```

如果需要固定版本，建议不要长期使用 `latest`，而是指定明确镜像标签。

## 回滚

如果升级后异常，优先回滚镜像版本和配置。

回滚前先确认数据库是否发生迁移。如果新版本已经修改数据库结构，直接回退旧版本可能出现兼容问题。

## 升级后检查

升级完成后检查：

- 后台能否登录；
- 渠道列表是否正常；
- 令牌是否可用；
- 请求日志是否正常记录；
- 客户端是否可以流式输出；
- 自定义前端是否还在。
