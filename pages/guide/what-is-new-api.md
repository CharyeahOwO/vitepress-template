# 什么是 New API

New API 是一个开源的统一 AI 模型网关。它可以把多个上游模型服务接入到同一个管理后台，再通过兼容 OpenAI 等常见协议的接口提供给客户端使用。

你可以把它理解成一层模型访问入口：客户端不需要关心背后具体接的是哪家服务商，只需要调用 New API 暴露出来的接口。

## 它解决什么问题

在真实使用里，模型来源通常不会只有一个：

- 不同厂商的模型能力不同；
- 不同渠道的稳定性和价格不同；
- 不同客户端支持的协议不同；
- 多个应用共用密钥时不好管理权限和额度；
- 上游故障时需要快速切换备用渠道。

New API 的作用就是把这些零散入口收拢起来，形成一个统一的管理层。

## 适合谁使用

New API 适合这些场景：

- 个人用户想统一管理多个模型接口；
- 团队内部需要给不同成员分配独立令牌；
- 开发者希望让自己的应用只对接一个模型网关；
- 需要记录请求日志、统计消耗、管理渠道可用性；
- 想把 OpenAI、Claude、Gemini、本地模型或其他兼容服务放到同一入口后面。

## 它不是什么

New API 本身不是模型厂商，也不凭空提供模型能力。它需要你配置可用的上游渠道。

New API 也不是万能的稳定性保证。上游不可用、密钥失效、网络异常、模型限流等问题，仍然需要结合日志和渠道状态排查。

## 基本链路

常见访问链路如下：

```text
客户端 / 应用
  ↓
New API 统一入口
  ↓
渠道配置与模型映射
  ↓
上游模型服务
```

对客户端来说，它通常只需要知道：

```text
Base URL
API Key
Model Name
```

其他复杂度交给 New API 后台处理。
