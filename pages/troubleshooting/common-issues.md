# 常见问题

这里整理 New API 使用中最常见的一些问题。

## 401 Unauthorized

通常是认证失败。

检查：

- API Key 是否填错；
- 是否使用了 New API 令牌，而不是上游 Key；
- 令牌是否被禁用；
- 客户端是否正确发送 `Authorization: Bearer`；
- Base URL 是否指向了正确的 New API 实例。

## 404 Not Found

常见原因：

- Base URL 写错；
- `/v1` 重复或缺失；
- 客户端请求了 New API 当前不支持的接口；
- 反向代理路径规则写反。

先看客户端实际请求 URL，再看反代日志。

## 模型不存在

检查：

- 客户端填写的模型名；
- 后台模型配置；
- 渠道支持模型；
- 模型映射；
- 分组权限。

## 请求超时

可能原因：

- 上游响应慢；
- 反向代理超时太短；
- 流式响应被缓冲；
- 网络链路不稳定；
- 模型上下文太长。

如果普通短请求正常，长请求失败，优先看超时和上下文长度。

## 流式输出不正常

表现可能是：

- 一次性吐出；
- 中途断开；
- 前端一直转圈；
- 客户端解析失败。

检查：

- 客户端是否启用 stream；
- 上游是否支持 stream；
- 反代是否关闭缓冲；
- CDN 是否缓存或改写响应；
- 返回格式是否与客户端兼容。

## 消耗异常

如果余额消耗明显不对，检查：

- 模型倍率；
- 渠道倍率；
- 是否有循环请求；
- 是否有共享令牌被滥用；
- 日志中的 token 统计是否合理。

## 后台打不开

检查：

- 容器是否运行；
- 端口是否监听；
- 反代是否指向正确端口；
- 数据库是否正常；
- 日志是否有启动错误。
