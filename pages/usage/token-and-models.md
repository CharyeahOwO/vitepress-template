# 令牌与模型

令牌和模型是客户端接入 New API 时最常接触的两个配置。

## 创建令牌

建议为每个客户端或应用单独创建令牌。例如：

- OpenClaw 一个令牌；
- 酒馆一个令牌；
- Claude Code 或其他开发工具一个令牌；
- 测试脚本一个令牌。

这样做的好处是：

- 日志来源清楚；
- 某个客户端异常时可以单独禁用；
- 额度和分组更好控制；
- 不需要频繁更换所有客户端配置。

## 配置模型

客户端请求时填写的 `model` 必须是 New API 后台可识别的模型名。

如果请求返回模型不存在，通常检查：

- 模型名是否拼错；
- 渠道是否支持该模型；
- 模型映射是否正确；
- 令牌所在分组是否有权限；
- 渠道是否启用。

## Base URL

OpenAI 兼容客户端通常填写：

```text
https://你的域名/v1
```

不要重复拼接 `/v1`。如果客户端已经自动补 `/v1`，就只填根地址；如果客户端要求完整 Base URL，就填到 `/v1`。

这个地方很容易写反，出问题先看客户端实际请求地址。

## API Key

API Key 填 New API 创建的令牌，不是上游服务商的 Key。

```text
客户端 → New API 令牌
New API → 上游服务商 Key
```

两者不要混用。

## 测试建议

配置完成后先用简单问题测试，再测试长文本、流式输出和多轮对话。

如果简单请求正常，长对话失败，通常和上下文长度、模型限制或反代超时有关。
